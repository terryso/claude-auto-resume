# Story 1.1: Auto-Resume Command Line Script for Claude Tasks

## Status: Done

## Story

**As a** user who needs to run long-running Claude tasks,  
**I want** an automated script that can automatically detect, wait, and resume tasks when encountering Claude CLI usage limits,  
**so that** I can efficiently complete tasks without manual intervention.

## Acceptance Criteria

1. Users only need to execute the script once to complete auto-resumption without manual intervention.
2. The script can correctly identify Claude CLI "usage limit" prompts and accurately extract recovery time.
3. The script can automatically calculate and wait until the recovery point based on system time.
4. After recovery, it can automatically continue tasks and output appropriate prompts.
5. If no limits are encountered, the script should exit directly and output "No waiting required".
6. The script only depends on standard Shell commands and claude CLI, requiring no additional dependencies.
7. Error scenarios (such as claude CLI exceptions, output format changes) have reasonable prompts or exit mechanisms.

## Dev Notes

- All logic is encapsulated in a single Shell script (claude-auto-resume.sh) with no other dependencies.
- Only depends on standard Unix commands (grep, date, sleep, etc.) and claude CLI.
- Related processes and architecture detailed in [docs/architecture.md], requirements in [docs/prd.md].
- No frontend needed, no cloud deployment required, runs completely locally.
- Error handling needs to cover CLI exceptions, output format changes, etc., to ensure script robustness.

### Testing

- Testing method is manual testing, complying with PRD requirements.
- Testing standard: Script can correctly handle different CLI output scenarios (with/without limits, abnormal output, etc.).
- Test files do not need to be stored separately, directly execute script in local command line and observe output.
