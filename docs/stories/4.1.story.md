## Dev Notes

### Architecture Context

**Technology Stack Migration**: Moving from shell script to TypeScript represents a significant architecture change while maintaining the core simplicity that users expect. The single-file philosophy transitions to a modular TypeScript structure while still producing a single executable output.

**Backward Compatibility**: This foundation must support eventual 100% feature parity with the current shell script (claude-auto-resume.sh:688). All existing command-line interfaces and usage patterns must be preserved.

**Performance Considerations**: The project must maintain <100ms startup time overhead compared to the shell script. This requires careful bundling optimization and efficient Node.js runtime integration.

### Implementation Strategy

**Incremental Development**: Start with minimal viable CLI structure and gradually add complexity. This allows for early validation of the build pipeline and ensures a solid foundation before implementing complex features.

**Testing Foundation**: Establish comprehensive testing infrastructure early to support test-driven development of core functionality migration in subsequent stories.

**Modular Architecture**: The project structure supports clean separation of concerns and makes the codebase maintainable as it grows to include all shell script functionality.

### Integration Points

**Build Pipeline**: The build system must integrate seamlessly with existing Makefile installation methods. Users should experience no change in installation workflow.

**Development Workflow**: TypeScript development environment should support rapid iteration and debugging while maintaining the robustness that production users require.

**CI/CD Integration**: Automated testing and building ensures quality consistency and supports the project's reliability requirements.
