# Story 4.2: Core Logic Migration

## Status: Done

## Story

**As a** developer working on the claude-auto-resume project,  
**I want** all existing shell script functionality migrated to TypeScript with proper type safety,  
**so that** the CLI application has the same capabilities as the original script with enhanced reliability and maintainability.

## Acceptance Criteria

1. **Argument parsing migrated to Commander.js with type-safe interfaces**: All command-line arguments and options from the shell script are properly parsed using Commander.js with comprehensive type definitions
2. **Environment variable handling with validation and defaults**: Environment variables (CLAUDE_AUTO_RESUME_WAIT_BUFFER, CLAUDE_AUTO_RESUME_SKIP_PERMISSIONS, CLAUDE_AUTO_RESUME_LOG_FILE) are validated and have proper defaults
3. **Claude CLI interaction module with proper error handling**: All Claude CLI interactions including timeout protection, output parsing, and error handling are implemented
4. **Time calculation utilities with cross-platform compatibility**: Date/time calculations work correctly on both Linux and macOS platforms
5. **Network connectivity checking with multiple fallback methods**: Network connectivity validation using ping, curl, wget with proper fallback mechanisms
6. **Custom command execution with security safeguards**: Custom command execution mode with proper warnings and security measures
7. **All error scenarios handled with structured error types**: Comprehensive error handling with specific error codes and helpful user messages
8. **Configuration validation with runtime type checking**: All configuration options are validated at runtime with proper error messages

## Dev Notes

### Previous Story Insights
Story 4.1 successfully established the TypeScript foundation with proper project structure, Commander.js integration, and build pipeline. The modular architecture is ready for core functionality migration. All development tooling (ESLint, Prettier, Jest, tsup) is configured and working.

### Core Functionality Analysis
The shell script (claude-auto-resume.sh) contains comprehensive functionality that must be migrated:

**Command-Line Interface**: 
- Options: -p/--prompt, -c/--continue, -e/--execute/--cmd, -h/--help, -v/--version, --check, --test-mode
- Positional arguments: custom prompt as default argument
- Complex validation preventing incompatible flag combinations

**Environment Variables**:
- CLAUDE_AUTO_RESUME_WAIT_BUFFER: Additional wait time (default: 0)
- CLAUDE_AUTO_RESUME_SKIP_PERMISSIONS: Control permission skipping (default: true)  
- CLAUDE_AUTO_RESUME_LOG_FILE: Optional logging destination (default: empty)

**Core Logic Flow**:
1. Environment variable validation and configuration loading
2. Argument parsing and validation
3. Claude CLI validation and network connectivity check
4. Claude command execution with timeout (30 seconds)
5. Usage limit detection and timestamp extraction
6. Wait time calculation with cross-platform date handling
7. Live countdown display with interrupt handling
8. Resume command execution or custom command execution

**Error Handling**: Comprehensive error scenarios with specific exit codes:
- Exit 1: Claude CLI execution failures
- Exit 2: Invalid timestamp extraction
- Exit 3: Network connectivity failures  
- Exit 4: Resume command failures
- Exit 5: Empty output scenarios
- Exit 130: User interruption (Ctrl+C)

### TypeScript Interfaces Required

```typescript
interface CLIOptions {
  prompt?: string;
  continue?: boolean;
  execute?: string;
  testMode?: number;
  version?: boolean;
  help?: boolean;
  check?: boolean;
}

interface Config {
  waitBuffer: number;
  skipPermissions: boolean;
  logFile?: string;
}

interface UsageLimitResult {
  hasLimit: boolean;
  resumeTimestamp?: number;
  rawOutput: string;
  waitSeconds?: number;
}
```

### Technical Constraints
- **Cross-platform compatibility**: Must work on Linux and macOS with different date command implementations
- **Timeout handling**: All Claude CLI interactions must have 30-second timeout protection
- **Signal handling**: Proper cleanup on SIGINT (Ctrl+C) and graceful termination
- **Version compatibility**: Support Node.js 18+ as defined in package.json
- **Performance**: Maintain <100ms startup time overhead requirement
- **Security**: Maintain security warnings for custom command execution

### Configuration Precedence
Configuration loading must follow this precedence order:
1. Command-line arguments (highest priority)
2. Environment variables
3. Default values (lowest priority)
