# Story 1.2: Environment Validation

## Status: Completed

## Story

**As a** user who runs claude-auto-resume,  
**I want** the tool to verify my environment and Claude CLI installation before proceeding,  
**so that** I can receive clear feedback about any setup issues and avoid runtime failures.

## Acceptance Criteria

1. Check if Claude CLI is installed and accessible in the system PATH
2. Verify that the Claude CLI version supports the `--dangerously-skip-permissions` flag
3. Display clear error messages when Claude CLI is not found or incompatible
4. Exit gracefully with appropriate error codes when environment validation fails
5. Show warning messages for potentially unsupported Claude CLI versions
6. Perform validation checks early in the script execution before any Claude commands

**Code Implementation**: [Source: docs/ROADMAP.md#environment-validation]
```bash
# Check if Claude CLI is installed
if ! command -v claude &> /dev/null; then
    echo "[ERROR] Claude CLI not found. Please install Claude CLI first."
    exit 1
fi

# Verify --dangerously-skip-permissions support
if ! claude --help | grep -q "dangerously-skip-permissions"; then
    echo "[WARNING] Your Claude CLI version may not support --dangerously-skip-permissions"
fi
```
